# xiaohui-meta-image 内容流水线工厂 v1.2

> 「告诉我你的赛道，我还你一条全自动流水线。」

**xiaohui-meta-image** 是一个面向 AI Agent 的**元 Skill（Meta-Skill）**。
用户只需要描述自己的内容赛道（如"小红书穿搭"、"公众号玄学"），它就能自动完成：赛道诊断 → 资产扫描 → 知识库匹配 → 生成完整流水线 Skill → 实际生图测试 → 可选 Cron 部署。

---

## ✨ 核心能力

| 能力 | 说明 |
|------|------|
| **赛道诊断** | 交互式追问，确认平台、垂类、受众、变现路径 |
| **资产扫描** | 自动扫描 6 大平台（Hermes/OpenClaw/Claude/Cursor/Codex/通用）已安装的 Skills |
| **语义理解** | 不只是扫目录名，还读取 SKILL.md description 做语义理解，判断实际能力 |
| **知识库匹配** | 内置 18 个「平台 × 垂类」最佳实践组合 |
| **流水线生成** | 输出完整可执行的 Skill（SKILL.md + references + scripts） |
| **测试验证** | 生成后必须先跑一遍完整流程（含实际生图），用户审核通过才部署 |
| **反馈记忆** | 用户反馈自动记录，下次生成同赛道时自动应用（越用越聪明） |

## 🚀 快速开始

### 安装到 Hermes Agent

```bash
git clone https://github.com/{your-username}/xiaohui-meta-image.git ~/.hermes/skills/xiaohui-meta-image
hermes skills reload
```

### 使用方式

直接在对话中输入：
- `帮我生成小红书穿搭的内容流水线`
- `我想做自媒体，不知道做什么`（进入推荐模式）
- `赛道匹配` / `内容工厂` / `小辉流水线`

## 🧠 工作流架构

```
Phase 0: 赛道诊断
  ├─ 0.1   接收输入（文字/截图/模糊需求）
  ├─ 0.1.5 资产扫描（6 平台路径 + 语义理解）
  ├─ 0.2   诊断追问（最多 2 轮）
  ├─ 0.2.5 个性化追问（按垂类动态触发）
  └─ 0.3   输出赛道画像（含个性化信息）

Phase 1: 知识库匹配
  ├─ 1.1   加载 track-knowledge-base.md（18 个组合）
  └─ 1.2   提取最佳实践

Phase 2: 流水线生成
  ├─ 2.1   创建 Skill 目录（按平台动态路径）
  ├─ 2.2   生成 SKILL.md（含历史反馈自动注入）
  ├─ 2.3   生成 references（style-guide + tag-library + content-calendar）
  ├─ 2.4   生成 templates（copywriting + image-prompt）
  ├─ 2.4.5 按需生成 scripts/data/config
  └─ 2.5   质量自检

Phase 2.5: 用户测试验证
  ├─ 2.5.1 触发完整测试（必须实际生图）
  ├─ 2.5.2 用户审核
  ├─ 2.5.3 迭代规则（最多 3 轮）
  └─ 2.5.4 反馈记忆（写入 feedback-log.md）

Phase 3: 可选 Cron 部署（仅测试通过后触发）
```

## 🛠️ 技术特性

### 跨平台资产扫描
不只是扫描目录名做关键词匹配，还会读取每个 skill 的 `SKILL.md` 前 5 行，提取 `description` 字段做语义理解。即使目录名是 `my-custom-tool`，也能识别出它是"八字排盘工具"。

### 生图 Fallback 链
```
优先级 1: image_generate 工具（FLUX 2 Pro）
  ↓ 失败
优先级 2: Nano Banana 2 中转 API
  ↓ 失败
优先级 3: Python Pillow 排版（terminal 预装依赖）
```

### 反馈记忆机制
用户反馈自动记录到 `feedback-log.md`，下次生成同赛道时自动应用。
- 用户说"这个调整不错"→ 标记为"全局应用"，影响所有后续生成
- 用户说"这个只对我有效"→ 标记为"局部应用"，只影响当前用户
