# xiaohui-meta-image 内容流水线工厂 v1.3.2

> 「告诉我你的赛道，我还你一条全自动流水线。」

**xiaohui-meta-image** 是一个面向 AI Agent 的**元 Skill（Meta-Skill）**。
用户只需要描述自己的内容赛道（如"小红书穿搭"、"公众号玄学"），它就能自动完成：赛道诊断 → 资产扫描 → 知识库匹配 → 生成完整流水线 Skill → 实际生图测试 → 半自动试运行 → 全自动部署。

---

## ✨ 核心能力

| 能力 | 说明 |
|------|------|
| **赛道诊断** | 交互式追问，确认平台、垂类、受众、变现路径 |
| **资产扫描** | 自动扫描 6 大平台（Hermes/OpenClaw/Claude/Cursor/Codex/通用）已安装的 Skills |
| **语义理解** | 不只是扫目录名，还读取 SKILL.md description 做语义理解，判断实际能力 |
| **知识库匹配** | 内置 18 个「平台 × 垂类」最佳实践组合（YAML 结构化，含正反示例） |
| **竞品标杆** | 每赛道 3 个标杆账号（共 54 个），提供风格锚点参考 |
| **流水线生成** | 输出完整可执行的 Skill（SKILL.md + references + scripts） |
| **测试验证** | 生成后必须先跑一遍完整流程（含实际生图），用户审核通过才部署 |
| **半自动缓冲** | 测试通过后默认 3 天半自动试运行，审核确认后才切全自动 |
| **反馈记忆** | 用户反馈自动记录，下次生成同赛道时自动应用（越用越聪明） |
| **反馈归档** | 90 天以上的 track 反馈自动归档，global 反馈永久保留 |

## 🚀 快速开始

### 安装到 Hermes Agent

```bash
git clone https://github.com/lizhaohuide-web/xiaohui-meta-image.git ~/.hermes/skills/xiaohui-meta-image
```

### 使用方式

直接在对话中输入：
- `帮我生成小红书穿搭的内容流水线`
- `我想做自媒体，不知道做什么`（进入推荐模式）
- 上传竞品截图 + `帮我分析这个账号的赛道`（截图诊断模式）

### 触发关键词

`内容流水线` / `赛道匹配` / `生成流水线` / `内容工厂` / `小辉流水线`

## 🧠 工作流架构

```
Phase 0: 赛道诊断
  ├─ 0.1   接收输入（文字/截图/模糊需求）
  ├─ 0.1.5 资产扫描（仅模糊需求时触发，6 平台 + 语义理解）
  ├─ 0.2   诊断追问（缺几个问几个，不变成问卷）
  ├─ 0.2.5 个性化追问（按垂类动态触发，最多 2 个问题）
  └─ 0.3   输出赛道画像

Phase 1: 知识库匹配
  ├─ 1.1   加载 track-knowledge-base.yaml（18 个组合）
  └─ 1.2   提取最佳实践（爆款公式/文案风格/配图规范/禁忌清单）

Phase 2: 流水线生成
  ├─ 2.1-2.4 创建 Skill 目录 + 生成全部文件
  └─ 2.5   质量自检

Phase 2.5: 用户测试验证（关键步骤）
  ├─ 2.5.1 触发完整测试（含实际生图，不能只给 Prompt）
  ├─ 2.5.2 用户审核 + 迭代（最多 3 轮）
  └─ 2.5.3 反馈记忆

Phase 3: 半自动试运行 → 全自动部署
  ├─ 3.1   半自动模式（默认 3 天，每天推送审核）
  └─ 3.2   确认后切换全自动 Cron
```

## 🛠️ 技术特性

### 跨平台资产扫描

不只是扫描目录名做关键词匹配，还会读取每个 skill 的 `SKILL.md` 前 5 行，提取 `description` 字段做语义理解。即使目录名是 `my-custom-tool`，也能识别出它是"八字排盘工具"。

支持平台：Hermes / OpenClaw / Claude Desktop / Cursor / Codex CLI / 通用目录

### 生图三级 Fallback 链

```
Level 1: Nano Banana 2 中转 API（AI 生图）
  ↓ 失败
Level 2: HTML + Puppeteer 渲染（现代卡片/信息图/金句，5 种主题配色）
  ↓ 失败
Level 3: Python Pillow 保底（主题感知渐变卡片）
```

支持 3 种样式：`--style card`（默认）/ `infographic` / `quote`

自动检测 5 种主题配色：育儿（暖橙）/ 科技（冷蓝）/ 玄学（深紫金）/ 美食（暖橙）/ 健身（活力绿）

### 选题采集三级数据源

```
Level 1: 6551 API - Twitter/X 热门推文（需 TWITTER_TOKEN）
Level 2: 6551 API - OpenNews 新闻聚合（需 OPENNEWS_TOKEN）
Level 3: Tavily Search API（需 TAVILY_API_KEY，免费 1000 次/月）
```

主数据源全部失败时自动降级到 Tavily 兜底。

### 反馈记忆 + 自动归档

- 用户反馈自动记录到 `feedback-log.json`
- `track` scope → 仅影响该赛道 | `global` scope → 影响所有赛道
- 超过 90 天的 track 反馈自动归档到 `feedback-archive.json`
- global 反馈永久保留
